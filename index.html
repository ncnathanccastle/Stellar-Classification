Prediction of Astrological Objects:
Star, Quasar, or Galaxy?

This project focuses on constructing several classification models to determine whether a given celestial object is a star, quasar, or galaxy. Initial steps include inspecting the given data and understanding each of the variables presented before cleaning and transforming the data into a form that will be suitable for model building. Using multiple different classification methods, the models will be tuned and compared across several scoring metrics including accuracy, f1-score, and run times.

The data used in this analysis was retreived from Kaggle which was originally extracted from the Sloan Digital Sky Survey

Project Goal:

Determine which model or models result in the most accurate classification in respect to every target class while taking into account the strengths and weaknesses of each model

Techniques highlighted in this project:

  • Several multiclass classification methodologies
  • Large dataset management
  • Outlier detection
  • Hyperparameter tuning
  • Cross validation
  • Feature selection

Class Imbalance¶

Class imbalance can sometimes have a large affect on the predictive ability of a classifier. In certain cases, a classifier will tend to classify the most frequent class well while the smaller class or classes tend to be overlooked and perform worse. There are certain resampling techniques, such as SMOTE or Tomek-Links, that can be implemented to help balance the class sizes and remedy this. However, there are still several models that are able to handle class sizes with different proportion of available data. In this case, the class sizes will remain unchanged.

Outlier Detection¶

Before beginning the more rigorous elements of building the classification models, outliers should be taken into account to avoid these instances from having an impact on the remaining analysis. Though the idea and definition of an outlier varies depending on the circumstances, there are methods that can be used to identify potential outliers. One such methodology is the Local Outlier Factor algorithm.

This algorithm works by calculating the local densities of a given data point and comparing it with respect to its nearest neighbors. Points that have substantially lower densities than that of their nearest neighbors are considered to be outliers. Removing outliers early before performing transformations or modeling can help improve the overall result of the finalized classifiers.

Some models require that the input variables have little to no correlation between them as base assumptions in order for the classifier to function properly and have interpretable results. Train-Test Split¶

When generating a model, one way to determine if the model is doing a sufficient job at classifying this particular type of data is by attempting to classify it using a brand new set of data that hasn't been used in any phase of modeling. To acheive this, the data can be split into two sets: One training set and one testing set.

The larger training set will be used to build and tune the model using cross validation while the smaller testing set will remain unaltered until the final classifiers have been selected and final scoring metrics are being evaluated.

Another reason to transform the data using principal component analysis has to do with the multicollinearity problem stated earlier. User-Defined Functions¶

Although standardization and principal component analysis were already performed in previous steps, the use of a pipeline will ensure that the scoring metrics calculated during cross validation will be as accurate as possible. If the standardized and PCA transformed dataframe is used as the input data for cross validation, the transform will have been calculated using the entire dataframe instead of each k-folds training set which could result in data leakage and incorrect scoring metric values. Since this will be done several times over all the candidate models, the use of user-defined functions will make this much easier to perform.

CV_Results¶

This function uses a pipeline which standardizes the input data, transforms it using principal component analysis, and performs stratified cross validation on several different metrics to evaluate the suitability of multiple different models for classification. This function returns a dataframe with each row corresponding to a different model and each columns corresponding to a different scoring metric. A pipeline must be used in order to ensure that for each fold in the cross validation set, the training set is the set of data that is being used to determine the standardization and principal component transforms while the testing set is the seperate set of data that is only being transformed using the values calculated on the training set. This leads to more robust and accurate scoring metrics being calculated.

  • x: The dependent variables or input
  • y: The independent variables or target
  • model: An array of classification models to be evaluated
  • pc: The number of principal components to retain while performing PCA
  • cv: The number of stratified folds to be evaluated during cross validation

Test_Results¶

This function uses a pipeline which standardizes the training input data, transforms it using principal component analysis techniques, fits the selected classification models, and measures the classification capabilites for each given model on the testing data by calculating several scoring metrics. This is similar to the earlier function CV_Results, but instead of performing cross validation it evaluates the performance of the model on the withheld testing data for the final results.

  • X_train: Training dataset dependent variables
  • X_test: Testing dataset dependent variables
  • y_train: Training dataset independent variables
  • y_test: Testing dataset independent variables
  • model: Array of classification models to be evaluated
  • pc: The number of principal components to retain while performing PCA

def CV_Results(x, y, model, pc, cv):
    
    #Set up dataframe that will be returned with corresponding column names
    display = pd.DataFrame(columns=['Model','Balanced Accuracy','Macro-precision','Macro_Recall',
                                    'Macro_F1','Fit Time','Prediction Time'])
    
    #Iterate cross validation scoring metrics for each model
    for i in model:
        scaler = StandardScaler()       #Standardization Step
        pca = PCA(n_components=pc)      #Principal Component Analysis step
        pipe = Pipeline(steps=[('scaler',scaler), ('pca',pca), ('model',i)]) #Set up pipeline including standardization and PCA
        skf = StratifiedKFold(n_splits=cv, shuffle=True, random_state=2022)  #Set up stratified k-folds for cross validation
        
        #Perform stratified cross validation using pipeline to calculate scoring metrics
        crvl = cross_validate(pipe, x, y, cv=skf, scoring=['balanced_accuracy','precision_macro','recall_macro','f1_macro'])
        
        #Gather model name and metrics to added to dataframe
        model    = type(i).__name__
        accuracy = round(crvl['test_balanced_accuracy'].mean(),4)
        mac_prec = round(crvl['test_precision_macro'].mean(),4)
        mac_rec  = round(crvl['test_recall_macro'].mean(),4)
        mac_f1   = round(crvl['test_f1_macro'].mean(),4)
        fit_time = round(crvl['fit_time'].mean(),4)
        pred_time= round(crvl['score_time'].mean(),4)
        
        model_metrics = [model, accuracy, mac_prec, mac_rec, mac_f1, fit_time, pred_time]
        
        #Add model metrics to returned dataframe
        display.loc[len(display)] = model_metrics
    #Remove index numbers for presentation
    display.index = ['']*len(display)
    
    return(display)

def Test_Results(X_train, X_test, y_train, y_test, model, pc):
        
    #Set up dataframe that will be returned with corresponding column names
    display = pd.DataFrame(columns=['Model','Balanced Accuracy','Macro-precision','Macro-recall',
                                    'Macro-F1','Fit Time','Prediction Time'])
    
    #Iterate cross validation scoring metrics for each model
    for i in model:
        scaler = StandardScaler()   #Standardization Step
        pca = PCA(n_components=pc)  #Principal Component Analysis Step
        pipe = Pipeline(steps=[('scaler',scaler), ('pca',pca), ('model',i)])  #Set up pipeline including standardization and PCA
        #Calculate fit time
        start = time()
        pipe.fit(X_train,y_train)
        end = time()
        fit_time = round(end-start,4)
        #Calculate prediction time
        start = time()
        predicted = pipe.predict(X_test)
        end = time()
        pred_time = round(end-start,4)
        #Gather model name and final scoring metrics to be added to dataframe
        model    = type(i).__name__
        balanced_accuracy = round(balanced_accuracy_score(y_test,predicted),4)
        precision = round(precision_score(y_test,predicted,average='macro'),4)
        recall = round(recall_score(y_test,predicted,average='macro'),4)
        f1 = round(f1_score(y_test,predicted,average='macro'),4)
        
        model_metrics = [model,balanced_accuracy,precision,recall,f1,fit_time,pred_time]
        #Add model final metrics to returned dataframe
        display.loc[len(display)] = model_metrics
    #Remove index numbers for presentation
    display.index = ['']*len(display)
    
    return(display)

Classification Modeling¶

There are several different modeling procedures that can be used for classification which each have their own strengths and weaknesses. When determining appropriate models to try, it is important to highlight key characteristics of the data that may have an effect on which models would work better than others.

Key Characteristics of the Data:

  • The response variable has highly imbalanced classes
  • The dataset is very large with about 100,000 instances
  • The input variables were transformed using principal component analysis which may introduce information loss
  • The principal components have low correlation values with the response variable so linearity assumptions may not hold

With these features in mind, five baseline models have been selected as a starting point for determining a final classification model.

Baseline Models¶

Logistic Regression¶

Logistic regression is a parametric supervised learning algorithm that is commonly used for classification. This statistical model is an example of a generalized linear model that uses the logit function as a link function to measure the log-odds of a given event thereby measuring the probability of a given outcome occurring. It uses maximum likelihood estimation and typically another iterative process, such as Newton's method, to estimate regression coefficients which provide an interpretable metric of each predictor variable's contribution to the overall probability of the outcome event.

Advantages:
Easily interpretable coefficients, quick prediction speeds, handles multinomial classification, straightforward and efficient model training

Disadvantages:
Affected by multicollinearity, dependent variables must be linearly related to the log-odds, linear decision surface limits classification capabilities

K-Nearest Neighbors¶

The k-nearest neighbors algorithm is a non-parametric supervised learning algorithm which can handle both classification and regression. This methodology stores all the feature vectors along with their corresponding outcome classes and performs prediction of a new input by comparing it to the closest k neighbors, typically determined from its Euclidean distance, before using a basic majority vote to determine the correct class. Due to it approach, k-nearest neighbors is referred to as a "lazy learning" since no real training happens other than retaining the feature vectors.

Advantages:
No training period, can be calculated using various distance metrics, flexible to different types of data relationships (linear or non-linear)

Disadvantages:
Difficultly with large datasets or datasets with many features due to the number of distance calculations required, sensitive to noisy data

Support Vector Machine¶

Another group of non-parametric supervised learning models are support vector machines. Support vector machines identify hyperplanes that maximize the distance of the hyperplane from the nearest point of differing classes creating a region. Using various different kernels, these hyperplanes can be linear or non-linear depending on what best divides the dataset into separate class regions. Support vector machines must use the training data to find an appropriate hyperplane from all the different instances and kernels, but once the hyperplane is identified prediction is usually much quicker since it just comes down to which class region the new data points fall under.

Advantages:
Works well in higher dimensions, kernels provide flexibility to data structures, memory efficient by only needing the final hyperplane

Disadvantages:
Costly for large datasets, many hyperparameters to optimize, susceptible to overfitting and noisy data, low interpretability

Random Forest¶

Random Forests are an ensemble supervised learning method which work by using a collection of decision trees whose results are aggregated into a the final output. Using bootstrap samples and random selections of the features, a forest of decision trees are generated and trained that will work as a collective to process new data and determine a final output, typically using the majority vote for classification or averages for regression. Since each tree is trained with a slightly different bootstrap sample and with slightly different input features, random forests can be robust and work well to prevent overfitting.

Advantages:
Prevents overfitting, less susceptible to outliers, able to handle large and high dimensional datasets, able to handle missing values

Disadvantages:
Provide feature importance but not entirely interpretable, many hyperparameters to optimize, better results require more decision trees be trained

Extreme Gradient Boosting¶

Gradient boosting is another form of ensemble learning where a weak learner, typically a decision tree, is trained, evaluated, and retrained using the results of the previous step to try and improve the model further by continually learning from its mistakes. Extreme gradient boosting is a variation of this where the objective function is regularized in order to further improve the outcome by working to prevent overfitting. As opposed to a random forest where a collection of decision trees are all fitted and aggregated for a final result, extreme gradient boosting creates one decision tree at a time and improves upon it to hopefully result in a single, more accurate, model.

Advantages:
Typically result in highly accuracy prediction, handle missing data, no pre-processing of the data is required, able to handle large and high dimensional datasets

Disadvantages:
Susceptible to outliers, continual boosting can lead to overfitting, many hyperparameter to optimize, may need to build many trees to converge

Validation Metrics¶

To compare the different classification models, various metrics will be used to determine the overall performace of each model as well as their performances in comparison to one another. Choosing the correct metrics requires determining what the overall end goal should be whether that is computational speed, overall accuracy, or some other aspect.

In this case, the goal is to build a classification model that is as accurate as possible by classifying all three response classes accurately and reducing the number of incorrect classifications.

Since the classes are highly imbalanced but are deemed equally as important to identify correctly, macro-average metrics of precision, recall and f1-score will be used so that each class is weighed equally as important. Also due to the imbalanced class sizes found in the data, a balanced accuracy score will be calculated for comparison. Although less important to the goal of having as accurate a model as possible, the fit times and prediction times will also be compared to give a more in-depth look as to the performance of each model.

Summary and Reflection¶

All three final models have approximently equivalent scoring metrics accounting for the overall classificiation performance, but the random forest model is the best by a margin of less than 1% across all scoring metrics.

The random forest model also has the longest fit time of any of the final models requiring nearly a minute to calculate while the k-nearest neighbors classifier has the shortest with both fit and prediction times under one second. The extreme gradient boosting model has the second longest fit time of around 45 seconds, but has the shortest prediction time of the final models with just over a tenth of a second to classify the new data.

One possible way to improve these models would be to use a resampling technique to better balance the three response class sizes. Some classification models such as logistic regression tend to perform better when there is more of a balance between the various classes. Since the dataset is rather large and contains thousands of instances from all the class, an undersampling technique such as condensed nearest neighbors, editted nearest neighbors, or even a combination of the two could also help to balance the difference in class sizes. These will strategically removing majority class instances that are misclassified based on its nearest neighbors thereby using a smaller portion of the data which will result in shorter fit and prediction times. Another idea could be to combine an undersampling and oversampling technique such as SMOTE-Tomek Links which would generate new artificial data points from the minority class while also removing borderline instances that may be misclassified based on its neighboring data points. Although this method may increase the overall size of the dataset, it could improve the overall classification capabilites by completely balancing the class sizes given prediction and fit times were of less importance to overall performance.

Since the three final models were flexible in regards to the relationship between the inputs and the output, another possibility to improve the results could be to remove performing principal component analysis from the pipeline. By removing this, there would be less of an information loss from the original variables to those used in the final modeling process which may work to improve classification capabilities.

A final suggestion would be to use a more powerful classification methodology such as an artificial neural network to handle this particular dataset. Although they offer less insight into the direct relation between specific inputs and their contribution to the classification process, artificial neural networks can handle much larger datasets with very little configuration adjustments which may be a good option if the overall goal is classification performance rather than identifying input-output relationship.
